# マルチプラットフォーム投稿文生成ツール 開発履歴

## 📋 プロジェクト概要

**目的**: 過去に書いたブログ記事（または参考にしたい記事）と新しいトピックをもとに、その人らしい文体でnote、LinkedIn、X（Twitter）用の投稿文を自動生成する。

**開発期間**: 2025年7月16日

---

## 🎯 仕様要求

### 📥 入力ファイル
- **文体ガイド**: `haru_writing_guide.md`
- **SNS投稿方針**: `sns_posting_workflow.md`
- **過去のnote記事群**: `note_md_files/`フォルダ内の.mdファイル群
- **テーマやキーワード**: 「営業の自動化について書きたい」など

### 📤 出力ファイル
- **note記事案**: 800〜1500文字
- **LinkedIn投稿案**: 300〜600文字
- **Twitter（X）投稿案**: 140文字×1〜3連投

## 🚀 使用方法

### CLI版
```bash
python main.py --styleguide input/haru_writing_guide.md --snsworkflow input/sns_posting_workflow.md --articles input/note_md_files/ --theme input/topic.txt
```

### WebUI版

- **課題**: 同一Wi-Fi内でのみアクセス可能
- **要求**: 外出先でもスマホから24時間アクセス可能

### ngrok導入
- **目的**: 外部からのアクセス実現
- **結果**: `https://******.ngrok-free.app` で外部アクセス成功
- **制限**: Pythonサーバー起動が必要

### 24時間稼働への挑戦

#### Railway クラウドデプロイ準備
1. **Procfile作成**:
   ```
   web: python web_app.py
   ```

2. **web_app.py修正**:
   ```python
   port = int(os.environ.get("PORT", 5000))
   debug_mode = os.environ.get("FLASK_ENV") != "production"
   app.run(debug=debug_mode, host='0.0.0.0', port=port)
   ```

3. **runtime.txt作成**:
   ```
   python-3.11.0
   ```

4. **requirements.txt確認**: 既存のままで対応可能

#### 予定されるデプロイフロー
1. GitHubリポジトリ作成・アップロード
2. Railway接続・自動デプロイ
3. 24時間稼働URL取得: `https://yourapp.railway.app`
